# Flutter 4.5：分享物料（文案复制 + 系统分享）v0.1-alpha 设计稿

日期：2026-04-27  
状态：已确认（用户“确认 Flutter 4.5 设计”）  
对应 WBS：4.5 分享物料（结果卡+文案复制）

---

## 1. 背景

Flutter 4.1–4.4 已完成：

- 匿名登录
- 录音 + 上传
- 推理调用 + 结果页渲染
- 问题库 Top3 列表/详情页

当前结果页已具备可读的结论与解释，但仍缺少“可传播”的能力，不满足 WBS 4.5 对分享闭环的要求。

WBS 4.5 的完整目标原本包括：

- 结果卡
- 文案复制

但在 v0.1-alpha 这一轮，为了优先打通“可分享”链路，先实现：

- 文案复制
- 系统分享

结果卡图片导出/保存延后到下一轮。

---

## 2. 目标（DoD）

1) 结果页可一键复制分享文案  
2) 结果页可调起系统分享面板  
3) 若当前结果关联 `problemId`，优先从后端模板接口拉分享模板  
4) 若模板接口不可用或失败，自动回退到客户端默认文案  
5) 默认分享文案包含 `#喵懂 #喵测`  

---

## 3. 用户流程

用户在 `ResultScreen` 看到分析结果后，可执行：

1) 点 **复制分享文案**
   - 客户端生成最终文案
   - 写入系统剪贴板
   - Snackbar 提示“已复制”

2) 点 **系统分享**
   - 客户端生成最终文案
   - 调起系统分享面板（微信/备忘录/短信等系统可用目标）

---

## 4. 分享文案来源策略

### 4.1 优先：后端模板接口

当 `problemId != null` 时，请求：

`GET /v1/templates/result?problem_id=<problemId>`

如果响应包含：

```json
{
  "share_asset": {
    "share_title": "...",
    "share_text_template": "我家猫这次更像：{label}（{confidence}%）…",
    "card_style": "result_card_v1"
  }
}
```

则优先使用：

- `share_text_template`

并做变量替换：

- `{label}`
- `{confidence}`

### 4.2 回退：客户端默认模板

若：

- 当前没有 `problemId`
- 或模板接口请求失败
- 或响应里没有 `share_text_template`

则使用本地默认模板：

```text
我家猫这次更像：{label}（{confidence}%）。来喵懂试试 #喵懂 #喵测
```

---

## 5. 变量替换规则

### 5.1 label

来源：

- `primary_intent.label`

若为空：

- 回退为 `未知`

### 5.2 confidence

来源：

- `primary_intent.confidence`

处理：

- 若为 0~1 浮点：转为百分比整数
- 若缺失：回退为 `-`

---

## 6. UI 改动（ResultScreen）

在结果页增加两个按钮：

- `复制分享文案`
- `系统分享`

按钮位置：

- 放在结果页底部操作区，与已有“有用/没用”同区域

反馈：

- 复制成功：Snackbar
- 分享调起失败：Snackbar

### 7.1 依赖

- `share_plus`：调用系统分享
- `flutter/services.dart`：使用 `Clipboard` 写剪贴板

### 7.2 代码落点

- `apps/mobile/pubspec.yaml`：新增 `share_plus`
- `apps/mobile/lib/screens/result_screen.dart`
- `apps/mobile/lib/api/api_client.dart`（如需直接复用 `getJson` 即可，无需额外改动）

- 不新增独立 share service，v0.1-alpha 可先把逻辑放在 `ResultScreen`
