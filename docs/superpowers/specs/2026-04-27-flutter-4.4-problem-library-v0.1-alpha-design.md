# Flutter 4.4：问题库列表/详情页（Top3）v0.1-alpha 设计稿

日期：2026-04-27  
状态：已确认（用户“确认 Flutter 4.4 设计”）  
对应 WBS：4.4 问题库列表/详情页

---

## 1. 背景

Flutter 4.1/4.2/4.3 已打通最小闭环：

- 匿名登录（token/user_id）
- 录音（10 秒）+ upload-url + PUT upload_url
- 调用 `POST /v1/inference` 并展示结构化结果页（degraded 兜底 + feedback）

当前 “问题库” Tab 仍为占位页，无法支撑 M2（v0.2-beta）所需的“Top3 问题库完整可用”。

---

## 2. 目标（DoD）

1) 问题库 Tab 可展示 **Top3**（或 limit=3）问题列表  
2) 可进入问题详情页，展示该 problem 的 bundle：
   - problem 基本信息（title/summary/tags）
   - questions 列表（精简展示）
   - suggestions 列表（精简展示）
   - tools_guide（默认折叠）
3) 详情页提供 **“开始喵测”**：
   - 进入录音页并携带 `problem_id`
   - 推理完成的结果页可以回跳到该问题详情

---

## 3. API 对接（与当前后端一致）

### 3.1 列表

`GET /v1/problems?limit=3`

响应：
```json
{
  "content_version": 2,
  "items": [
    { "id":"night_meow", "title":"...", "summary":"..." }
  ]
}
```

### 3.2 详情

`GET /v1/problems/{id}`

响应为 **bundleResp**：`content_version` + bundle 字段（bundle 是“内联展开”，并非 `bundle:{...}`）：

```json
{
  "content_version": 2,
  "problem": {...},
  "questions": [...],
  "suggestions": [...],
  "tools_guide": {...}
}
```

### 3.3 鉴权与灰度

客户端需携带：

- `Authorization: Bearer <token>`

以便后端按 user_id 做 content_version 灰度（可与结果页/问题库保持一致的版本体验）。

## 4. 页面结构

### 4.1 ProblemsScreen（Tab2）

内容：

- 顶部：显示 `content_version`
- 列表：卡片展示 `title/summary`
- 点击列表项 → push `ProblemDetailScreen(problemId)`

错误态：

- 网络失败 / 解析失败：显示错误文案 + “重试”按钮

### 4.2 ProblemDetailScreen

内容结构：

1) Header：title + tags（如有）+ summary
2) 追问（Questions）：列表展示 `text` 与 `type`（options 仅在 choice 时展示）
3) 建议（Suggestions）：列表展示 `title` 与前 1-2 条 step
4) 工具区（Tools Guide）：默认折叠（collapsed_by_default）
5) 底部 CTA：开始喵测 → 进入录音页并带上 `problemId`

---

## 5. 与 4.2/4.3 的衔接

### 5.1 RecordScreen

新增可选参数：

- `problemId`（nullable）

进入录音页时若携带 problemId：

- UI 上显示 `problem_id`
- 推理调用时继续把 `problemId` 传给结果页（用于回跳）

### 5.2 ResultScreen

新增可选参数：

- `problemId`（nullable）

若存在：

- 增加按钮“查看问题详情”→ push `ProblemDetailScreen(problemId)`

降级态下：

- “去问题库”可先保持 snackbar（v0.1）或后续演进为切换 tab
