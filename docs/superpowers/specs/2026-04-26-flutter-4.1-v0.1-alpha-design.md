## 1. 背景

当前仓库中 `apps/mobile` 仍是预留目录，仅有 README，无可运行 Flutter 工程。  
后端已具备：

- `POST /v1/auth/anonymous`：返回 `user_id` 与 `token`（MVP token 形态为 `dev-token-<user_id>`）
- 其它核心 API（内容读取、推理、反馈等）已逐步就绪

WBS M1（v0.1-alpha）的目标之一是“端到端最小闭环可跑通”，因此 Flutter 工程底座与导航必须先稳定。

---

## 2. 目标（v0.1-alpha）

1) `apps/mobile` 变为可运行 Flutter 工程（iOS/Android 可启动）  
2) 基础导航与页面壳（先占位，便于 4.2/4.3 迭代）：
   - 底部导航：**喵测 / 问题库 / 我的（占位）**
   - 喵测页：入口按钮跳转录音页（录音页先占位）
   - 问题库页：占位
   - 我的页：显示 user_id、提供清缓存/退出（占位）
3) 启动时自动获取匿名身份（若本地无 token）并缓存 `user_id/token`，后续请求可复用

非目标（v0.1-alpha 不做）：

- 不做真正录音/上传/推理调用（在 4.2/4.3）
- 不做复杂路由（例如 deep link、多级栈管理）
- 不做 UI 视觉精修（保证结构清晰即可）

## 3. 页面与导航结构

### 3.1 AppShell（底部导航）

- `Scaffold`
  - `body`: IndexedStack/切换屏
  - `bottomNavigationBar`: 3 tabs

Tabs：

1) 喵测：`HomeScreen`
2) 问题库：`ProblemsScreen`
3) 我的：`MeScreen`

### 3.2 HomeScreen（喵测入口）

- 顶部标题 + 简短说明
- 主按钮：进入 `RecordScreen`（占位页）
- 次按钮：进入问题库 tab（或 push 问题库页，v0.1 先切 tab）

### 3.3 RecordScreen（占位）

仅展示：

- “录音页（施工中）”
- “下一步会实现：10 秒上限、m4a、上传拿 audio_id”

### 3.4 ProblemsScreen（占位）

仅展示：

- “问题库（施工中）”
- “下一步会对接 /v1/problems 列表与详情”

### 3.5 MeScreen（占位 + 可用信息）

- 展示：
  - `user_id`
  - token 是否存在（不展示完整 token，最多展示前缀/长度）
- 操作：
  - “清除本地缓存”（清 token/user_id，并提示下次会重新匿名登录）

## 4. 网络层与身份缓存

### 4.1 ApiClient（轻量封装）

- 基于 `package:http`（或 Flutter 自带 HttpClient 也可，v0.1 选 http 更简单）
- 统一 baseUrl（开发期可用 `http://127.0.0.1:8080`，后续再抽配置）
- 自动注入 header：
  - `Authorization: Bearer <token>`（若已登录）
  - `Content-Type: application/json`

### 4.2 本地存储

使用轻量本地持久化（v0.1 可用 shared_preferences）存：

- `user_id`
- `token`

启动逻辑：

1) 若本地已有 token：跳过登录
2) 若无 token：调用 `POST /v1/auth/anonymous`，缓存 `user_id/token`

失败策略（v0.1）：

- 登录失败：在 HomeScreen 显示错误提示，并提供“重试”按钮

```

    main.dart

      api_client.dart
    storage/
```

- 每个文件单一职责
- 先把 “壳” 固化，后续 4.2/4.3 直接在对应 screen 内逐步填充
