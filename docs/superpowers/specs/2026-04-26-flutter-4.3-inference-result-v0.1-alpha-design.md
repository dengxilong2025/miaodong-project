# Flutter 4.3：调用推理 + 结果页渲染（结构稳定）v0.1-alpha 设计稿

日期：2026-04-26  
状态：已确认（用户“确认 Flutter 4.3 设计”）  
对应 WBS：4.3 调用推理 + 结果页渲染（结构稳定）

---

## 1. 背景

Flutter 4.1 已完成工程骨架与匿名登录；Flutter 4.2 已实现录音 + 上传（`/v1/audio/upload-url` + `PUT upload_url`）。

后端当前推理编排接口为：

- `POST /v1/inference`
  - 请求：`{ "audio_url": "...", "context": {...} }`
  - 成功响应：在推理服务输出基础上补齐 `request_id/content_version/degraded=false/inference_latency_ms`
  - 失败响应：`degraded=true` 且包含 `degraded_reason/message/fallback`

推理服务（stub）当前输出最小字段：

- `schema_version/model_version/primary_intent/explanations`

因此移动端 v0.1-alpha 需要做到：

- “能打通链路 + 结果页结构稳定”
- “降级可用（不要崩）”
- “最小反馈闭环”

---

## 2. 目标（DoD）

1) 上传完成后，能够用 `audio_url` 调用 `POST /v1/inference`  
2) 结果页展示结构稳定（至少）：
   - `primary_intent.label` + `confidence`
   - `explanations[]`（factor + text）
   - 调试信息（可折叠）：`request_id/content_version/inference_latency_ms`
3) 降级（`degraded=true`）有明确兜底：
   - 展示 `message` 与 `degraded_reason`
   - 提供“重试”与“去问题库”
4) 最小反馈闭环：
   - “有用/没用”按钮调用 `POST /v1/feedback`（至少传 `request_id/helpful`；可选传 problem_id/content_version）

---

## 3. 关键数据流（与 4.2 衔接）

### 3.1 从录音页进入分析

4.2 上传成功时，后端 `upload-url` 返回：

- `audio_id`
- `audio_url`（用于推理服务读取/联调下载）

移动端在 4.2 中需要保存并传递：

- `audioUrl`（必填，用于 /v1/inference）
- `audioId`（可选，用于 UI 展示/调试）

### 3.2 推理请求体（v0.1-alpha）

调用：

`POST /v1/inference`

body：

```json
{
  "audio_url": "https://.../a_xxx",
  "context": { "time_of_day": "night|day" }
}
```

其中 `time_of_day` v0.1-alpha 采用“自动推断”：

- 22:00–06:00 → `night`
- 其它 → `day`

> v0.2 再扩展为更丰富的 context 表单（例如 before_feeding 等）。

---

## 4. 结果页 UI（最小但结构稳定）

页面：`ResultScreen`

### 4.1 顶部结论卡

- 标题：更像：`primary_intent.label`
- 副标题：confidence 百分比
- 若 `degraded=true`：显示“暂时不可用”样式

### 4.2 解释区（为什么这么判断）

列表渲染 `explanations[]`：

- `factor` 作为小标签
- `text` 作为正文

### 4.3 操作区

- “有用 / 没用”
  - 调 `POST /v1/feedback`
  - 成功 toast/snackbar
- “再测一次”（返回录音页/重新开始录音流程）
- “去问题库”（切到问题库 tab；v0.1 可先提示占位）

### 4.4 调试信息（折叠）

折叠区展示：

- request_id
- content_version
- inference_latency_ms
- degraded_reason（如有）

---

## 5. 降级策略（degraded=true）

当响应包含 `degraded=true`：

- 主文案：展示后端 `message`
- 辅助：展示 `degraded_reason`（timeout/unreachable/bad_status/bad_json）
- 按钮：
  - 重试：重新请求 /v1/inference（保持同样 audio_url/context）
  - 去问题库：切换到问题库 tab
