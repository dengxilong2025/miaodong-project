# Flutter 4.6 Retest Compare + Achievement v0.1-alpha Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 Flutter 客户端实现同一 `problemId` 的复测对比卡、基线结果本地持久化、`POST /v1/retest` 提交，以及首个客户端成就“初次复测”的最小闭环。

**Architecture:** 复用现有 `LocalStore` 做本地持久化；新增 `retest/retest_store.dart` 负责 baseline 与成就状态；在 `ResultScreen` 成功态根据 `problemId + request_id` 判断是否已有 baseline，并展示“保存为基线 / 对比卡 / 提交复测 / 保存为新的基线”；提交成功后解锁本地成就并弹出对话框。

**Tech Stack:** Flutter (Dart) + shared_preferences（已有）+ http（已有）。
