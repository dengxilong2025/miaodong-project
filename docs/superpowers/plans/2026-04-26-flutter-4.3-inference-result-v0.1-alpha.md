# Flutter 4.3 Inference + Result v0.1-alpha Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 Flutter 客户端实现“上传后调用 `/v1/inference` + 结果页渲染 + degraded 兜底 + 最小反馈按钮”，形成 v0.1-alpha 的端到端闭环：录音→上传→推理→结果页（可重试/可反馈）。

**Architecture:** 在 `apps/mobile` 新增 `ResultScreen` 与轻量 `InferenceClient`（复用 ApiClient.postJson）；`RecordScreen` 上传成功后保留 `audioUrl/audioId` 并启用“开始分析”按钮；ResultScreen 内部管理 loading/success/degraded/error，展示结构化字段并提供 feedback（/v1/feedback）。

**Tech Stack:** Flutter (Dart) + http（已在 ApiClient）+ shared_preferences（已有）。

## Task 1: InferenceClient（调用 /v1/inference）

**Files:**
- Create: `apps/mobile/lib/inference/inference_client.dart`

- [ ] **Step 1: 定义响应解析（保持宽松）**

v0.1-alpha 以 Map 透传，避免过早强类型导致频繁改动：

```dart
class InferenceClient {
  InferenceClient({required this.api});
  final ApiClient api;

  Future<Map<String, dynamic>> infer({
    required String audioUrl,
    required Map<String, dynamic> context,
  }) async {
    return api.postJson('/v1/inference', {
      'audio_url': audioUrl,
      'context': context,
    });
  }
}
```

- [ ] **Step 2: 自动推断 context.time_of_day**

```dart
String timeOfDay() {
  final h = DateTime.now().hour;
  return (h >= 22 || h < 6) ? 'night' : 'day';
}
```


- [ ] **Step 3: 状态机**

页面状态：
- loading
- success（degraded=false）
- degraded（degraded=true）
- error（网络/解析异常）

将完整 response Map 保存在 state，用于调试折叠区显示。

- [ ] **Step 4: UI 渲染（最小但结构稳定）**

- 重试按钮（重新调用 infer）
- 去问题库（v0.1：SnackBar 提示；或后续接 tab 控制）

调试折叠区（ExpansionTile）：
- request_id
- content_version
- inference_latency_ms
