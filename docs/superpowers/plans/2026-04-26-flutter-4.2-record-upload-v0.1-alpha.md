# Flutter 4.2 Record + Upload v0.1-alpha Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 Flutter 客户端实现“录音（10 秒上限）+ 上传（upload-url + PUT upload_url）”，兼容后端返回的 `storage=api-direct|minio-presign`，并在 UI 中展示上传进度与 `audio_id`，为 4.3 推理调用提供输入。

**Architecture:** 新增 `recording/recorder.dart` 封装录音 start/stop/limit；新增 `upload/uploader.dart` 封装 `POST /v1/audio/upload-url` + `PUT upload_url`；`RecordScreen` 作为状态机（idle/recording/recorded/uploading/uploaded），按钮显式触发；错误用 SnackBar/文本提示。

**Tech Stack:** Flutter (Dart) + `record` + `permission_handler` + `http`。

## Task 2: Recorder 封装（10 秒上限）

**Files:**
- Create: `apps/mobile/lib/recording/recorder.dart`

- [ ] **Step 1: 实现 Recorder**

```dart
enum RecorderState { idle, recording }

class Recorder {
  Future<void> init();                 // 权限检查/请求
  Future<String> start({required Duration maxDuration}); // 返回 filePath
  Future<void> stop();
  RecorderState get state;
}
```

实现要点：
- start 时创建临时文件路径（`path_provider.getTemporaryDirectory()`）
- start 时传入 `maxDuration: Duration(seconds:10)`（若插件不支持则用 Timer stop 兜底）

- [ ] **Step 2: 在 ApiClient 新增 putBytes 辅助**


判定：
- 200/204 成功
- 其它状态码抛异常（带 body）

- [ ] **Step 3: 实现 Uploader**

```dart
class Uploader {
  Uploader({required this.api});
  final ApiClient api;

  Future<UploadInstruction> getUploadUrl({String? contentType, int? durationMs});
  Future<void> uploadBytes(UploadInstruction ins, List<int> bytes);
}
```

`getUploadUrl`：
- `POST /v1/audio/upload-url`，body 可先 `{}` 或带 `content_type/duration_ms`（后端当前不强校验）

`uploadBytes`：
- `PUT ins.uploadUrl`，headers 使用 `ins.headers`
