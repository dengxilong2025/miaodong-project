# ADR-0002：内容后台与版本化发布（1-2人轻流程）

日期：2026-04-22  
状态：Accepted

## 背景

产品大量体验来自“问题库/追问/建议/解释/分享模板/工具区”等内容与策略，且需要频繁迭代。团队规模 1-2 人，但必须避免线上口碑因内容误改而受损。

## 决策

采用“权威内容 + 运行时策略”两层模型，并配套版本化发布：

- 权威内容（Postgres）：problem/question/suggestion/risk/tools 等
- 运行时策略（配置）：解释模板、分享模板、排序与展示策略
- 发布能力：`draft → published → archived` 状态机；发布生成 `content_version`；支持灰度与一键回滚
- 线上记录：每次推理结果记录 `content_version`，可追溯

## 备选方案

1) 全写死在代码里：迭代慢，必须发版  
2) 全用 JSON 文件：统计/关联/审计与后台化困难  

## 后果

- 正向：运营可改内容；策略可灰度/AB；线上可回滚止损  
- 负向：需要实现“发布中心/版本/灰度/回滚”的最小系统能力
