# ADR-0003：增长玩法模块优先级与工程落地顺序（喵懂/喵测）

日期：2026-04-22  
状态：Accepted

## 背景

冷启动基本决定成败。需要在“解决痛点（留存）”之外，具备可炫耀/可晒/可连载的兴趣点支撑爆炸式传播，且不能牺牲科学克制与可解释性。

## 决策

增长模块的传播优先级（用户优先级）：
1) 喵懂人格卡  
2) 喵懂成就系统  
3) 猫咪日程热力图  
4) 猫猫昵称/称号  
5) 城市/同龄猫对比（后置，样本量与合规充分后启用）

工程实现顺序建议（成本/依赖最优）：
成就系统 → 人格卡 → 热力图 → 称号 → 同城对比。

## 原则

- 所有“炫耀”必须有证据（解释因子/统计聚合），避免玄学化
- 健康风险不出现在分享卡上（share_safe=false），避免恐慌式传播
- 复测与反馈为核心闭环：可连载、可对比，形成黏性护城河
